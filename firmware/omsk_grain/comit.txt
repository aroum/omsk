feat(ui): EG bipolar modulation UI overhaul & logical fixes

* **OLED UI (src/ui/oled_ui.cpp)**
  - Re‑implemented EG modulation bar drawing:
    - EG+ now fills **left‑to‑right**; EG‑ fills **right‑to‑left**.
    - Updated signed‑mod calculation to `float signed_mod = (mod_amt - 0.5f) * 2.0f;`.
    - Added explicit label handling (`EG+` / `EG-`) and kept other sources (`---`, `JIT`, `LF1`, `LF2`) unchanged.
    - Consolidated modulation amount, source detection and label rendering into a single block to avoid duplication.
    - Added safety clamping for fill width (`fill_w`) to stay inside the 30‑pixel bar.
  - Cleaned up unused variables and comments; removed stale “centered‑bipolar” logic.
  - Updated includes (`#include <cstdio>`).

* **MIDI CC handling (src/midi/midi_cc.cpp)**
  - Preserve previous modulation source to detect when EG is first assigned.
  - When EG is newly assigned, initialise `mod_amt` to the neutral centre (`0.5f`) instead of jumping to `+100%`.
  - Existing delta‑based amount adjustment kept for subsequent encoder moves.
  - Adjusted min‑mod logic to keep the original range (`-1.0f` for EG, `0.0f` for other sources).

* **Audio Engine (src/synth/audio_engine.cpp)**
  - Applied bipolar mapping for EG source (`‑1 … +1`) in `GranularEngine::get_modulated_param`.
  - Fixed JIT pitch quantisation: `SEMI` and `OCT` modes now round to whole semitones / octaves.
  - Minor refactoring of parameter‑ratio helpers and display strings (e.g., precise position display, density formatting).

* **Additional non‑code assets**
  - Numerous generated audio files in `audio2h/converted/` were touched by the build script; they are now staged.
  - Minor documentation updates in `docs/README.md` and `docs/problem.md`.
  - Minor formatting touch‑up in `src/config.h`.

* **Why**
  - The original implementation inverted the relationship between encoder direction and EG modulation, causing the bar to shrink when turning right. The new logic aligns physical encoder movement with visual feedback:
    - Turning **right** → `EG+` intensity grows, bar fills from left.
    - Turning **left** (through zero) → `EG‑` intensity grows, bar fills from right.
  - Initial EG assignment now starts at the neutral centre (0 % modulation) as required by the UI spec.
  - Cleaned up duplicated code paths and ensured consistent label handling across all modulation sources.

  - `src/synth/audio_engine.cpp`

  - `src/config.h`
